# NexaBudget Backend - Project Context & Instructions

This document provides foundational context and operational guidelines for the NexaBudget backend project.

## Project Overview

NexaBudget is a personal finance management application. This backend service is a RESTful API built with **Java 25** and **Spring Boot 4.x**. It integrates advanced features like Open Banking (GoCardless), crypto portfolio tracking (Binance), and AI-driven insights (Google Gemini).

### Core Stack
- **Framework:** Spring Boot 4.0.5 (Java 25)
- **Data Persistence:** 
  - **PostgreSQL:** Primary relational store for users, accounts, transactions, etc.
  - **MongoDB Atlas:** Vector store for AI semantic caching and embeddings.
- **Caching:** Valkey/Redis (via Redisson) for performance and job tracking.
- **Security:** Spring Security with JWT (stateless) and API Keys (M2M).
- **AI Integration:** Spring AI with Google Gemini (Categorization, Reports).
- **Architecture:** Layered (Controller -> Service -> Repository -> Model) with DTOs.
- **Deployment:** Docker & GraalVM Native Image support.

## Key Technical Conventions

### 1. Data Integrity & Persistence
- **Primary Keys:** Always use `UUID` (`@GeneratedValue(strategy = GenerationType.UUID)`).
- **Soft Delete:** `Transaction` and `Account` entities use soft deletion.
  - Managed via `@SQLRestriction("deleted = false")`.
  - Soft-deleted items are retained for 30 days before being purged by a scheduled job.
- **Audit Logging:** Automatically handled via `AuditAspect` (AOP). Every write operation on core entities is recorded in `audit_logs`.
- **Database Migrations:** DDL mode is set to `validate` in production; schema changes require manual migration scripts.

### 2. Security
- **Authentication:** Dual-mode support:
  - **JWT:** For user sessions (24h default expiry).
  - **API Keys:** For machine-to-machine access (SHA-256 hashed in DB).
- **Password Protection:** Always use `UserService.updateUserProfile()` for password changes to ensure BCrypt encoding. Never set `passwordHash` directly.
- **Ownership:** Always verify entity ownership (e.g., `getBudgetByIdAndUser`) before updates or deletions.

### 3. AI & Integrations
- **AI Categorization:** Transactions are categorized using `gemini-2.5-flash-lite`.
- **AI Reports:** Asynchronous financial analysis. Reports are generated by passing CSV data as multipart attachments to Gemini.
- **GoCardless Sync:** Uses atomic DB-level locks (`isSynchronizing` flag) to prevent race conditions during transaction sync.
- **Multi-Currency:** Automatic exchange rate retrieval and amount conversion for transfers between accounts with different currencies.

### 4. Performance & Resilience
- **Virtual Threads:** Enabled by default (`spring.threads.virtual.enabled=true`).
- **Retries:** Spring Retry is applied to GoCardless and Exchange Rate API calls.
- **Caching:** 6-hour TTL for bank data; 5-minute TTL for crypto prices.

## Building and Running

### Prerequisites
- JDK 25+
- Docker & Docker Compose
- Maven

### Key Commands
| Action | Command |
| :--- | :--- |
| **Build Project** | `./mvnw clean install` |
| **Run Locally** | `./mvnw spring-boot:run` |
| **Run Tests** | `./mvnw test` |
| **Native Build** | `./mvnw clean package -Pnative` |
| **Start Infra** | `docker compose up -d` |

## Repository Structure

- `src/main/java/it/iacovelli/nexabudgetbe/`
  - `config/`: Configuration (Security, AI, Cache, AOP).
  - `controller/`: REST endpoints (17 specialized controllers).
  - `dto/`: API request/response contracts.
  - `model/`: JPA entities and Enums.
  - `repository/`: Spring Data JPA/MongoDB repositories.
  - `security/`: JWT and API Key filters/providers.
  - `service/`: Core business logic and integrations.
- `src/main/resources/`: Application properties and SQL migrations.
- `k8s/`: Kubernetes deployment manifests.

## Development Guidelines

- **Surgical Edits:** When modifying services, ensure you don't break existing AOP auditing or soft-delete logic.
- **Validation:** Always include tests for new features. Use `./mvnw test` to verify changes.
- **Environment:** Ensure critical environment variables (`JWT_SECRET`, `GEMINI_API_KEY`, `CRYPTO_ENCRYPTION_KEY`) are provided. The application will fail to start if `JWT_SECRET` is insecure.
